## 1. Figma -> NPM배포까지 이어지는 파이프라인 구축

최종 파이프라인 결정

- Figma에서 디자인 시스템 수정 및 json 추출 (tokens studio 이용)
- `tokens/src/tokens.json` 파일 교체
- 빌드 시 style-dictionary 라이브러리를 이용해서 json -> css 평탄화 후 push
- main에 merge될 때 tokens.json 파일에 변경사항이 있다면 version up & npm 자동 배포 빌드

고민: 현재 dev에 바로 push하고 main을 배포 브랜치로 사용하고 있는데, 배포 전 테스트를 로컬에서 밖에 못한다는 단점이 있었습니다. 현재는 다음 근거에 판단하여 이 프로세스를 유지하기로 결정했습니다.

1. 현재는 혼자 개발을 하는 단계에서 한번 더 브랜치를 거치는 과정보다 로컬 테스트로 충분한다고 생각
2. Component는 Storybook 테스트 가능, design token은 자주 바뀌지 않는 특성이기에 현재 프로세스를 유지해도 괜찮다는 생각
